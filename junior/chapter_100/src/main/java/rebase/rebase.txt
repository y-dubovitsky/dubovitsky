# one more

Бывает ситуация, когда мы делаем нашу задачу не с первого раза. Исправляем или изменяем код.

В это случае в нашем репозитории наступает каша.

Давайте посмотри наш лог.

git log -4

Как мы видим два последних коммита касаются одной задачи. Эти коммиты надо объединить в один.

Для этого мы будем использовать интерактивный режим.

git rebase -i HEAD~3

HEAD~3 указывает, что мы хотим взять последних три коммита.

Теперь нажимаем кнопку i = входим в режим редактирования.

Вместо слово pick нужно написать слово squash.
