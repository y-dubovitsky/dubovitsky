Практике командной разработки, есть подход который обеспечивает совместный контроль качества исходного кода - peer review. При нем, каждый из участников команды проводит code review (проверяет код), написанный другим участником. Это помогает выявлять ошибки/несоответствия в коде при оценке "свежим взглядом".

В прилагаемых файлах исходный код. Ваша задача - провести оценку написанного кода.

Оценивайте всё, что считаете необходимым:

1) наличие ошибок

2) именование

3) нарушения правил оформления кода (code convention)

4) комментарии и т.д.

Продемонстрируйте ваши знания best practices. По каждому пункту замечаний напишите краткое обоснование.

Задание.

1. Создайте задачу.

2. В задачу в комментарии текстом отпишите, какие вы видите замечания.
